PRODUIT ACTIF
-------------
  EventViewer.WinUI  (WPF = legacy gelé)

COMPILATION
-----------
  USB :
    > build_winui_usb.bat
    → bin\WinUI_USB\EventViewer.WinUI.exe

  Store-flagged (sans maintenance) :
    > build_winui_store.bat
    → bin\WinUI_Store\EventViewer.WinUI.exe

  Dev :
    > dotnet build EventViewer.WinUI\EventViewer.WinUI.csproj -c Release -p:Platform=x64
    > dotnet run --project EventViewer.WinUI\EventViewer.WinUI.csproj -c Release -p:Platform=x64

UTILISATION
-----------
  1. Lancer en administrateur (recommandé pour les journaux Windows)
  2. Choisir le journal : Système / Applications / Sécurité
  3. Lire le bandeau santé, sélectionner un incident
  4. Suivre « Que faire » dans le panneau Comprendre

FONCTIONNALITÉS CLÉS
--------------------
  • Santé + timeline 24 h
  • Explications en français
  • Recherche + export CSV/JSON
  • Snapshot / nouveautés
  • Feedback Utile / Pas utile
  • Télémétrie opt-in (locale)
  • IA bêta optionnelle (local ou cloud si configuré)
  • Maintenance DNS/SFC/réseau (USB uniquement, pas Store)

IA CLOUD (OPTIONNEL)
--------------------
  1. Activer IA bêta dans Options
  2. Endpoint OpenAI-compatible
  3. Clé : variable EVENTVIEWER_AI_API_KEY
  Sans clé → reste 100 % local
